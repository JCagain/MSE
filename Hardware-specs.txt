# Implementation
(Regularly)
    ESP32                          Controller(done)
reads sensor      --routing-->     calculate route
confirm recieve  <--path_push--     send direction
                  --path_ack-->         

(When help button is pressed)
    ESP32                          Controller(done)
button pushed     --distress-->    confirm receive
reflect endpoint <--distress_ack--

All communication at 115200 baud over USB serial. 
One JSON object per line.

# Serial protocol example

**You send → controller:**

```json
// Periodic sensor report (send every few seconds)
{
  "type": "routing",
  "node_id": "1A",
  "is_passable": true,
  "temperature": 24.5,
  "co2": 0.12,
  "sensor_error": false
}

// Reply to every path_push
{"type": "path_ack"}

// When help button is pressed
//`seq` should increment by 1 on each button press so the controller can detect duplicates.
{
  "type": "distress",
  "node_id": "1A",
  "seq": 1,
  "floor": 1,
  "location_label": "Main Corridor West",
  "timestamp_ms": 1234567890
}
```

**Controller sends → you:**

```json
// Pointing direction
{"type": "path_push", "node_id": "1A", "next_hop_id": "1B", "path_distance": 7.5}

// Acknowledgement of your distress button press
{"type": "distress_ack", "node_id": "1A", "seq": 1, "timestamp_ms": 1234567890}
```
