# Design: MEGA + Laptop Split

**Date:** 2026-04-04
**Status:** Approved

## Overview

Split the current monolithic Java controller into two separate components:

1. **Arduino Mega** — safety-critical real-time controller: receives routing packets from the
   ESP32 gateway, runs multi-source Dijkstra, broadcasts `path_push` back to the mesh, and
   streams state snapshots to the laptop.
2. **Laptop** — passive display: reads state snapshots from the Mega over USB, drives
   `SwingDashboard`, and handles distress alerts (display only — no SMS/HTTP).

If the laptop is off, the Mega continues routing independently.

---

## Architecture

```
ESP32 mesh nodes
      │  routing / distress / heartbeat_ack  (newline-JSON)
      ▼
  ESP32 gateway  (single node)
      │  UART — Serial1 (115200 baud)
      ▼
 Arduino MEGA
  ┌─────────────────────────────────────┐
  │  • Parse routing / distress packets │
  │  • Node-timeout watchdog            │
  │  • Multi-source Dijkstra (C)        │
  │  • Broadcast path_push  → Serial1   │
  │  • Send state_snapshot  → Serial0   │
  │  • Forward distress     → Serial0   │
  │  • Relay distress_ack   ← Serial0   │
  └─────────────────────────────────────┘
      │  USB — Serial0 (newline-JSON)
      ▼
   Laptop
  ┌─────────────────────────────────────┐
  │  MegaBridge  (reads USB serial)     │
  │  SwingDashboard  (unchanged)        │
  │  DistressHandler (display only)     │
  └─────────────────────────────────────┘
```

## Section 1: MEGA Firmware

### Feasibility

- O(N²) array-scan Dijkstra (no heap) runs in microseconds at 16 MHz for N ≤ 50 nodes.
- Memory: ~200 bytes for distance array + 50 bytes for previous-node indices — well within 8 KB SRAM.
- ArduinoJson with a static document handles JSON parsing with zero heap fragmentation.

### File structure

```
mega/
├── mega.ino              # setup() / loop()
├── topology.h            # graph hard-coded as C arrays (generated by TopologyCompiler)
├── dijkstra.cpp/.h       # O(N²) multi-source Dijkstra
├── node_state.cpp/.h     # NodeState struct array + update from routing packet
├── serial_mesh.cpp/.h    # Serial1: read routing/distress, write path_push/heartbeat
└── serial_laptop.cpp/.h  # Serial0: write state_snapshot, forward distress, read distress_ack
```

### Main loop

```
setup():
  Serial.begin(115200)   // Serial0 → laptop
  Serial1.begin(115200)  // Serial1 → ESP32 gateway
  initNodeStates()

loop():
  if packet available on Serial1:
    parse with ArduinoJson (static doc, 512 bytes)
    if routing   → updateNodeState(), set recomputeNeeded = true
    if distress  → forward verbatim to Serial0
  if recomputeNeeded AND debounce elapsed (broadcastIntervalMs):
    runDijkstra()
    sendStateSnapshot()   // Serial0
    broadcastPathPush()   // Serial1
    recomputeNeeded = false
  if distress_ack received on Serial0:
    relay to Serial1
  checkNodeTimeouts()     // mark timed-out nodes impassable, set recomputeNeeded
```

### Topology representation (`topology.h`)

Generated at build time by `TopologyCompiler`. Not written by hand.

```c
#define NUM_NODES 4
#define MAX_NEIGHBORS 4

const char* NODE_IDS[]  = {"1A", "1B", "1C", "EXIT1"};
const bool  IS_EXIT[]   = {false, false, false, true};
const int8_t NEIGHBOR_IDX[NUM_NODES][MAX_NEIGHBORS] = {
    {1, -1, -1, -1},   // 1A → 1B
    {0,  2,  3, -1},   // 1B → 1A, 1C, EXIT1
    {1,  3, -1, -1},   // 1C → 1B, EXIT1
    {1,  2, -1, -1},   // EXIT1 → 1B, 1C
};
const float WEIGHTS[NUM_NODES][MAX_NEIGHBORS] = {
    {5.0f, 0,    0,    0},
    {5.0f, 3.0f, 4.0f, 0},
    {3.0f, 2.0f, 0,    0},
    {4.0f, 2.0f, 0,    0},
};
```

## Section 2: MEGA ↔ Laptop USB Protocol

Newline-delimited JSON on Serial0. Three packet types only.

### MEGA → Laptop: `state_snapshot`

Sent after every Dijkstra run.

```json
{
  "type": "state_snapshot",
  "nodes": [
    {
      "id": "1A", "floor": 1, "label": "Main Corridor West",
      "passable": true, "temp": 25.1, "co2": 0.12,
      "next_hop": "EXIT1", "dist": 5.0,
      "distress": false, "timed_out": false
    }
  ]
}
```

### MEGA → Laptop: `distress`

Forwarded verbatim from the mesh.

```json
{
  "type": "distress", "node_id": "1A", "seq": 3,
  "floor": 1, "location_label": "Main Corridor West",
  "timestamp_ms": 1712345678
}
```

### Laptop → MEGA: `distress_ack`

MEGA relays this to the ESP32 mesh via Serial1.

```json
{
  "type": "distress_ack", "node_id": "1A", "seq": 3,
  "timestamp_ms": 1712345679
}
```

## Section 3: Laptop-Side Changes

### Deleted

| Class | Reason |
|---|---|
| `Controller` | Replaced by `MegaBridge` |
| `SerialBridge` | MEGA owns the ESP32 serial connection |
| `HeartbeatService` | MEGA handles gateway heartbeat |
| `PathComputationService` | MEGA runs Dijkstra |
| `Simulator`, `SimNode`, `ScenarioRunner` | No longer applicable |

### Unchanged

| Class | Notes |
|---|---|
| `DistressRecord` | Unchanged value object |
| `Node`, `Exit`, `Graph`, `PathCandidate` | Unchanged domain model |
| `TopologyLoader` | Still used to build `Graph` for the dashboard node-graph view |

### Minor change: `SwingDashboard`

Currently holds a `Controller` reference directly. Change its constructor to accept a
`DashboardDataSource` interface instead:

```java
interface DashboardDataSource {
    Map<String, NodeState> getNodeStates();
    Graph getGraph();
    List<DistressRecord> getRecentDistressEvents();
}
```

`MegaBridge` implements this interface. No other changes to `SwingDashboard`.

### New class: `MegaBridge`

Entry point for laptop-side. Opens USB serial to the Mega (jSerialComm), reads packets in a
background thread, maintains `NodeState` map and distress record list, drives the dashboard.

```
MegaBridge
  fields:
    Graph graph                        // loaded from topology.json at startup
    Map<String, NodeState> nodeStates  // updated on each state_snapshot
    List<DistressRecord> distressLog   // appended on each distress packet
    SwingDashboard dashboard

  background thread:
    on state_snapshot → update nodeStates, dashboard.push()
    on distress       → append DistressRecord, send distress_ack, dashboard.push()

  public:
    getNodeStates()           // called by SwingDashboard (same as before)
    getGraph()                // called by SwingDashboard (same as before)
    getRecentDistressEvents() // called by SwingDashboard (same as before)
```

### Simplified `DistressHandler`

Reduced to a thread-safe list with a `handle(DistressRecord)` append method.
No Twilio, no HTTP, no retry queue, no file logging. Distress alerts appear in the
existing dashboard alerts panel.

## Section 4: TopologyCompiler

New CLI in the existing Maven project, alongside `TopologyValidator` and `TopologyGenerator`.
Reads `topology.json` via the existing `TopologyLoader`, assigns each node a numeric index,
and emits a `topology.h` C header.

```bash
# Run once whenever topology changes
java -cp target/mse-controller-1.0-SNAPSHOT.jar \
     mse.topology.TopologyCompiler sample-topology.json mega/topology.h
```

Output: `topology.h` with `NUM_NODES`, `MAX_NEIGHBORS`, `NODE_IDS[]`, `IS_EXIT[]`,
`NEIGHBOR_IDX[][]`, `WEIGHTS[][]` as shown in Section 1.

Re-flash the Mega after regenerating `topology.h`.

---

## Out of Scope

- Cellular fallback (already flagged as not implemented)
- Multiple gateway nodes (single gateway by design)
- Simulator (removed — test with real or bench hardware)
- SMS / HTTP distress notifications (display only for now)
